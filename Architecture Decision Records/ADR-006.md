# ADR-006: Метрики

## Контекст
Из-за дисбаланса классов и иерархии нужно выбрать метрику, которая оценивает работу модели на всём каталоге

## Варианты
1. **Accuracy.** Завышает оценку при дисбалансе 
2. **F1-weighted.** Учитывает дисбаланс, но всё равно доминируют топ классы.
3. **F1-macro.** Усредняет F1 по классам с равным весом - наказывает за плохую работу на хвосте.
## Решение
**Основная метрика:** F1-macro на child (47 классов).

**Дополнительные:**
- F1-weighted и accuracy - для сравнения и контекста.
- F1-macro на parent (9 классов) - через child -> parent 
  маппинг, оценка "ошибается ли модель внутри parent или 
  между parent".
- Per-class F1 - для error analysis и поиска проблемных 
  классов.

## Обоснование
- F1-macro равномерно оценивает все классы + соответствует использованию в паре с  class weights.

## Риски
- F1-macro может быть низким даже на хорошей модели, если 
  есть классы с малым количеством данных.
