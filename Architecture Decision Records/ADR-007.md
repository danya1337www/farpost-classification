# ADR-007: Preprocessing текста

## Контекст
У классических моделей и BERT разные требования к чистоте текста. Единый preprocessing для всех уровней будет либо недостаточен (для TF-IDF), либо избыточен 
(для BERT).

## Решение
**Вход:** title + ". " + description. param_1-3 не 
используются в текстовых моделях (отданы CatBoost как 
табличные фичи).
Пропущенный description заменяется пустой строкой, тк Nan, None в description могут сломать модели 

**Level 0 (TF-IDF + LinearSVC):**
- стоп-слова русского языка из NLTK
- min_df=5
- остальное TF-IDF делает сам

**Level 1 (CatBoost):**
- text_features, CatBoost обрабатывает дальше сам (не реализовано)
- категориальные фичи - fillna()

**Level 2 (BERT):**
- минимальная чистка: удаление лишних пробелов
- токенизация - токенайзером модели

## Обоснование
- TF-IDF будет проще от глубокой чистки, тк лемматизация склеивает "айфон/айфона/айфоном" в один токен. (не реализовано)
- CatBoost text_features - сделает все сам. (не реализовано)
- BERT обучен на сыром тексте с пунктуацией - остальное испортит результат

## Out of scope
- Лемматизация через pymorphy3 (рассматривалась, не реализована).
- CatBoost - text features
