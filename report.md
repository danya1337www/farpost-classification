# Классификация объявлений

## Задача

Решение для автоматической классификации объявлений и товаров по категориям.

## Данные

В качестве источника взят [Avito Demand Prediction Challenge](https://www.kaggle.com/c/avito-demand-prediction) с Kaggle. Это ~1.5M размеченных объявлений Avito с готовой иерархией категорий: 9 parent -> 47 child.

Использованные поля:

- `title`, `description` - текст объявления
- `parent_category_name`, `category_name` - иерархия категорий
- `param_1`, `param_2`, `param_3` - категориальные параметры товара
- `price`, `region`, `city`, `user_type` - метаданные

### EDA

- **Сильный дисбаланс на child-уровне.** Топ-5 классов = 53% всех объявлений. Самый частый класс "Одежда, обувь, аксессуары" (282k), самый редкий "Недвижимость за рубежом" (117). Соотношение ~2400x.
- **Дисбаланс на parent-уровне умеренный** (~38x): "Личные вещи" 697k vs "Для бизнеса" 18k.
- **Тексты:** title короткие и однородные (median 20 символов), description с длинным хвостом (median 96, max 3212), ~7.7% объявлений с пустым description.

## Подход

Решение построено как лестница из трёх моделей с постепенным усложнением.

### Подготовка данных

Из 1.5M объявлений сделан стратифицированный сэмпл 300k с сохранением распределения категорий. Сэмпл разбит на train/val/test в пропорции 80/10/10 со стратификацией по child-классу. Стратификация обязательна, тк без неё редкие классы могут не попасть в val или test.

### Препроцессинг

Текстовый вход: `title + ". " + description`. Пропущенный description заменяется пустой строкой. Параметры param_1-3 не объединяются с текстом - идут в CatBoost как табличные фичи.

Чистка текста минимальная: только удаление лишних пробелов. 
Дальнейшая обработка делается моделями: TfidfVectorizer сам приводит к нижнему регистру, токенизирует и убирает стоп-слова; 
BERT работает с сырым текстом.

### Уровни лестницы

**Level 0: TF-IDF + LinearSVC.** Текстовый baseline. TfidfVectorizer с русскими стоп-словами (NLTK), `min_df=5` для отсечения опечаток и редких слов, `token_pattern` оставляет только русские буквы. LinearSVC с `class_weight='balanced'` для компенсации дисбаланса.

**Level 1: CatBoost.** На вход: `region`, `city`, `user_type`, `param_1-3` как категориальные, `price` как числовая, `auto_class_weights='Balanced'`, `early_stopping_rounds=25` на val F1.

**Level 2: rubert-tiny2 + fine-tuning.** Flat классификация на 47 классов через HuggingFace Trainer. Кастомный `WeightedTrainer` для class weights в CrossEntropyLoss. 5 эпох, batch 64, learning rate 3e-5, warmup 10%, weight decay 0.01. `load_best_model_at_end=True` для выбора лучшего чекпоинта по F1-macro.

### Метрика

Основная - **F1-macro**. Усредняет F1 по классам с равным весом, что соответствует цели - равномерное качество по всему каталогу, не только на популярных категориях. F1-weighted используется как дополнительная.

## Результаты

| Модель | F1-macro | F1-weighted | Доля ошибок |
|---|---|---|---|
| Level 0: TF-IDF + LinearSVC | 0.8463 | 0.8944 | 10.6% |
| Level 1: CatBoost | 0.8384 | 0.9472 | 5.6% |
| Level 2: rubert-tiny2 | 0.8477 | 0.8747 | 12.7% |

### Главное

**Лучшая модель зависит от метрики.** BERT даёт лучший F1-macro - равномернее работает на редких классах. CatBoost даёт лучший F1-weighted и наименьшее число ошибок за счёт сильного сигнала от price/region/params на популярных категориях, но проигрывает на хвосте.

Где важна общая доля правильных предсказаний, лучше CatBoost. Для системы, где важна одинаковая работа по всем категориям каталога лучше BERT.

### Анализ ошибок

| Модель | Ошибок | Внутри parent | Между parent |
|---|---|---|---|
| TF-IDF | 3179 | 47% | 53% |
| CatBoost | 1686 | 27% | 73% |
| BERT | 3810 | 49% | 51% |

CatBoost ошибается реже, но если ошибается - то фундаментально (между parent). Табличные фичи дают сильный сигнал об общей категории, но при промахе модель может уйти в совсем другую ветку.

BERT и TF-IDF делят ошибки поровну между типами - ошибки внутри одной ветки иерархии, между близкими категориями.

### Перекрытие категорий

Самая частая пара ошибок BERT - **Одежда <-> Детская одежда** (~1000 случаев из 3810 ошибок) => Перекрытие категорий

Часть из ошибок - спорная разметка.
Модель формально ошибается, но предсказание не хуже истинного лейбла
### Per-class анализ для BERT

Идеальное качество (F1 = 1.0): Автомобили, Квартиры, Комнаты, Гаражи, Земельные участки, Дома и дачи. Это категории с уникальными ключевыми словами и однозначным контекстом.

Худшие классы (F1 < 0.65):

- Охота и рыбалка (F1=0.57) - мало данных, нечёткие границы со "Спорт и отдых"
- Оборудование для бизнеса (F1=0.59) - пересекается с "Ремонт и строительство"

## Подводные камни и решения

**Перекрытие категорий.** Часть ошибок семантическое пересечение категорий. Решение в проде - мульти-лейбл классификация

**Дисбаланс классов.** Соотношение между самым частым и самым редким классом ~2400x. Применены class weights во всех моделях. Для классов с очень малым числом примеров (Недвижимость за рубежом, 117 объявлений) это всё равно мало - F1 на них будет низким независимо от модели.

**Иерархия каталога.** Hierarchical классификация в "полном ее смысле"(к примеру multi-task с двумя головами) была выбрана в ADR, но не реализована в рамках ресурсов/бюджета. Вместо неё - flat классификация на child + two-level evaluation: метрики считаются и на child (47 классов), и на parent через маппинг.

**Шум в текстах.** Опечатки/сокращения/ошибки отсекаются `min_df=5` в TF-IDF. BERT справляется с шумом за счёт внутренней subword токенизации.

## Возможные улучшения

1. **Ансамбль моделей.** CatBoost + BERT с late fusion. Использовать сильные стороны обоих.
2. **Multi-task BERT** с двумя головами parent + child. 
3. **Мультимодальность.**  картинки + табличные + текст
4. **Тюнинг параметров** через Optuna для каждой модели - ожидаемый прирост 1-3% F1.
5. **BERT.** multilingual-e5-base или rubert-base - ёмкость в 6-9 раз больше tiny2, должен быть прирост F1-macro.

## Стек
